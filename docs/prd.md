# 需求文档

## 1. 应用概述

- **应用名称**：张明峰的个人主页
- **应用描述**：一个简约科技风的个人主页，面向面试官、朋友与家人，展示张明峰的个人信息，并提供一个可交互的数字分身聊天区，访客可通过聊天了解张明峰的基本情况与个性。

---

## 2. 用户与使用场景

### 2.1 目标用户
- 面试官：快速了解求职者背景与特点
- 朋友与家人：了解张明峰近况与个人信息

### 2.2 核心使用场景
- 面试官访问主页，浏览个人信息后，通过数字分身聊天区提问，快速了解求职者的优缺点、技能方向等
- 朋友或家人访问主页，查看个人介绍，与数字分身互动

---

## 3. 页面结构与核心功能

### 3.1 页面总览

```
个人主页（单页面，纵向滚动）
├── 顶部导航栏
├── Hero 区（头像 + 姓名 + 一句话介绍）
├── 个人信息展示区
└── 数字分身聊天区
```

### 3.2 顶部导航栏
- 固定在页面顶部，背景半透明白色，滚动时保持可见
- 左侧显示姓名「张明峰」作为 Logo 文字
- 右侧提供锚点导航链接：关于我、数字分身
- 支持点击锚点平滑滚动至对应区域
- 适配手机端：导航链接在小屏幕下折叠为汉堡菜单

### 3.3 Hero 区
- 居中布局，页面首屏展示
- 展示内容：
  - 头像（圆形裁切，预留头像占位图）
  - 姓名：张明峰
  - 一句话介绍：我是一个软件工程专业大四应届毕业生
- 视觉风格：淡蓝色渐变背景或科技感粒子/线条装饰（轻量实现，不影响性能）

### 3.4 个人信息展示区（关于我）
- 以卡片或分块形式展示以下信息：
  - **现在主要在做**：私企和央国企的春招准备
  - **兴趣方向**：AI 开发
  - **有记忆点的特点**：会吹笛子
- 布局简洁，图标 + 文字组合，适配手机端单列展示

### 3.5 数字分身聊天区
- 区域标题：和我的数字分身聊聊
- 聊天界面布局：
  - 上方为对话消息列表区域（可滚动）
  - 下方为输入框 + 发送按钮
- 数字分身初始欢迎语（页面加载后自动展示）：
  - 示例：「你好！我是张明峰的数字分身，你可以问我任何关于他的问题 :)」
- 用户输入问题后，数字分身基于以下预设知识库进行回复：
  - **职业/身份**：正在求职的大学生
  - **最近在做的事**：准备春招的笔试和面试，以及毕业设计
  - **擅长/关心方向**：关心 AI 的最新发展，以及如何用 AI 做出有用的产品
  - **高频问题预设回答**：
    - 你有什么优点？→ 预设回答（由系统基于上述信息生成合理默认回复）
    - 你有什么缺点？→ 预设回答
    - 你擅长什么？→ 预设回答
- 数字分身回复方式：调用大语言模型 API，以张明峰第一人称视角回答，语气自然、简洁
- 消息气泡样式：用户消息靠右（淡蓝色背景），分身消息靠左（白色/浅灰背景）
- 输入框支持回车键发送
- 发送后输入框自动清空，消息列表自动滚动至最新消息
- 适配手机端：聊天区高度自适应，输入框固定在聊天区底部

---

## 4. 业务规则与逻辑

- 数字分身聊天区调用大语言模型 API，系统提示词（System Prompt）中注入张明峰的个人背景信息，确保回复内容与人设一致
- 数字分身始终以第一人称「我」回答，不暴露「我是 AI」等系统信息
- 用户发送空消息时，不触发请求，输入框轻微抖动提示
- 数字分身回复期间，发送按钮置灰，显示加载状态（如省略号动画）

---

## 5. 异常与边界情况

| 场景 | 处理方式 |
|------|----------|
| 大语言模型 API 请求超时或失败 | 显示提示：「网络开小差了，请稍后再试」，不影响页面其他区域 |
| 用户连续快速发送消息 | 上一条回复未返回前，发送按钮保持置灰，防止重复请求 |
| 消息内容过长 | 气泡内自动换行，不超出容器宽度 |
| 手机端键盘弹出 | 输入框随键盘上移，聊天列表区域自适应压缩 |

---

## 6. 验收标准

- 页面在 PC 端与手机端均可正常访问与展示
- Hero 区头像、姓名、一句话介绍正确显示
- 个人信息展示区三项内容（在做的事、兴趣、特点）完整展示
- 数字分身聊天区可正常发送消息并收到回复
- 数字分身回复内容与张明峰人设一致，使用第一人称
- 顶部导航锚点点击后可平滑滚动至对应区域
- 整体视觉风格为淡蓝色 + 白色，简约科技感
- 手机端布局无错位，聊天输入框可正常使用
